## 3DGS

- 베이스라인 3DGS 모델로 LongSplat 선택, WeatherGS 모델에서 제공하는 데이터셋을 사용해 테스트 진행  
- 눈, 비 추가됐을 때 뒤에 사물 일그러지는 강도가 심해짐  
- 원본 렌더링 → 날씨 아티팩트 → 날씨 아티팩트 + 렌더링 순으로 성능 저하됨  

- 그러나 factory만 원본보다 날씨 아티팩트 추가한 데이터셋이 성능 더 좋게 나옴  

- raw dataset을 렌더링한 output이 심하게 망가져있는 상태이므로 결과를 분석할 수 있는 상태가 아님  
  → 실험이 제대로 진행된 게 맞는지 확인해볼 것, 세팅이 정상적이라면 LongSplat 모델 자체가 문제가 있는 것  

- 기본 3DGS 모델은 인풋으로 사진 + camera pose까지 필요한데 LongSplat은 이미지 넣으면 자동으로 camera pose까지 내부에서 추정해서 3DGS 진행  

- LongSplat 말고 기존에 잘 돌아가는 3DGS 모델을 사용하는 게 좋을 것 같다  

- 실험결과는 rain rendered와 raw rendered의 성능이 얼마나 차이나는지만 알 수 있으면 됨  
  → 이 gap을 어떻게 줄일 수 있을지 고민할 것

## 3DOD - snow

- PointPillars, PointRCNN, VoxelRCNN, MVMM 네 가지 모델 KITTI&KITTI_snow에 대해 eval 진행  

- 2D-BBox, BEV와 같은 2차원 metric에서 Pedestrian의 AP 측정 시 KITTI_snow가 KITTI보다 더 높게 나오는 경우가 존재했음  

- 보행자 박스가 작기 때문에 3D-Bbox, AOS AP 값이 상대적으로 낮게 나올 수 있음  

- 보행자가 정말 날씨에 대한 영향을 안 받는 건지 분석해보고 이유 생각해볼 것  
  → 진짜라면 이 자체가 논문의 좋은 contribution이 될 수 있다  

- 전체적인 결과는 예상대로이고, 3D 박스 시각화 자료를 통해 날씨에 따른 성능 저하를 면밀히 분석해볼 필요가 있다

## 3DOD - rain&night

- 5가지 논문의 컨셉 자체는 다 좋음  

- ContextualFusion이 weight까지 공개되어있는 유일한 논문이지만 MoME가 제일 최신(CVPR2025) 모델이므로 학습을 시도해보는 게 좋을 것 같다  

- 입력마다 전문가 모델을 골라서 처리하는 구조  
  → 그러나 transformer 모델 3가지 정도로 무게가 크지 않아 학습이 가능할 것 같다  

- 그러나 해당 논문의 결과를 봤을 때 condition, 특수한 환경에서 SOTA를 주장했기 때문에 실제 날씨 조건에서 SOTA인지는 확인해봐야 함  

- 논문 주제가 weather에 대한 고려가 있는 것이 아니고 악천후에서 좋은 결과가 나왔다는 내용이 존재하는 정도  

- 입력이 멀티뷰 이미지인데 weather 합성이 가능한지가 문제이다  

- 일단 MoME를 좀 더 자세히 분석해보는 게 좋을 것 같다
