## 3DOD_Snow - case study

### 진행 내용
- eval, 시각화 수행
- 눈 시뮬레이션 적용 데이터 분석

### 모델 성능 비교
- 성능 우수: Voxel R-CNN, MVMM
- 성능 저하: PointPillars, PointRCNN

### 관찰 결과
- 눈 시뮬레이션 적용 시 RiDAR로부터 먼 곳에서 성능 저하 확인
- 원인으로 눈 영향 추정

### 이전 실험 피드백
- 눈 시뮬레이션 적용 데이터셋에서 지표가 높았던 원인 검토
- Pedestrian label(BEV, 2D-BBox)은 Z축을 고려하지 않아 bounding box overlap 영향으로 추정
- 3D-BBox에서 MVMM 모델의 성능 차이가 컸던 이유는 카메라에 적용된 눈 노이즈 영향으로 추정

### 정진우 박사님 피드백
- 사람 scene의 경우에만 눈 시뮬레이션 적용 시 3DOD 성능이 크게 떨어지지 않음
- 사람 scene만의 특징을 분석하여 자동차와 사이클리스트에 적용하는 scheme 제안
- 이를 통해 다른 scene의 성능 개선 가능성 검토

## 3DOD_Night&Rain

### 현재 진행
- nuScenes 데이터셋으로 MoME 모델 성능 평가 진행

### 실험 결과
- night: 이상 레이블 제외 시 mAP 0.712 (기존 0.610)
- rain: 결과는 평이한 수준

### 연구 방향성
- MoMA 모델이 취약점을 보이는 night 환경에서 3DOD 성능 개선
- 전처리 적용: 이미지 밝기 보정, 어두운 영역 노이즈 필터링
- 전처리 한계 시: 밤 환경 특화 피처 추출 모듈 사용
- 맑은 날 데이터를 활용한 KD 기법 고려

### 모델 비교 자료
- night 기준 ContextualFusion의 성능이 가장 높음 (mAP 73.0)
- 야간에는 카메라 정보 손실이 커서, 야간 context를 입력받아 카메라 가중치를 낮추고 LiDAR 가중치를 높게 활용한 것으로 해석

### 정진우 박사님 피드백
- MoME는 Fusion 모델(camera + RiDAR)이며, camera / RiDAR / Fusion의 3개 전문가 구조
- weight에 초점을 두는 아이디어 제안
- 3DOD에서 영상 품질 개선만으로는 차별성이 부족할 수 있음
- 추가 아이디어 보완 권장
- MoMA가 SOTA이므로 모델은 확정

## 3DGS

### 현재 진행
- Free 데이터셋 Weather-GS 테스트 수행
- 프레임 수가 많아 이전 실험보다 성능 향상 확인
- 모델 자체 문제보다 데이터셋 특성 영향으로 판단
- Main figure 제작 및 논문 작성 진행

### 정진우 박사님 피드백
- 현 연구의 차별성을 더 명확히 하여 설득력을 높일 필요
- 차별성 요소:
  - SfM 처리 유무 (posed / unposed)
  - 전처리 시점 (재구성 기준)
- 하나의 조건을 정하거나, 두 조건을 모두 포함할 경우 다른 조건은 고정한 상태에서 실험할 것을 권장

### 비교 구도
- Weather-GS:
  - SfM 처리 완료
  - posed
  - 재구성 이전 처리: AEF (Atmospheric Effect Filter)
  - 이전과 이후 처리: LED (Lens Effect Detector)
- ours:
  - SfM 처리 없음(추정)
  - unposed
  - 이전 처리: 2D Refinement

### 가설 및 해석
- 데이터셋 간 프레임 수 차이가 성능 차이의 원인일 가능성 존재
- 가설: LongSplat(SfM 처리 없음)은 시점이 다양하지 않으면 잘 동작하지 않음
