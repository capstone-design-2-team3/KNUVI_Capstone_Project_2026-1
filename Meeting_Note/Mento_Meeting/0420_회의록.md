## 3DOD in Snow

### 준비한 내용

차량, 자전거 클래스에서 포인트 유실률이 높아 포인트 증강 실험 진행  

- 복셀 기준으로 나누어 포인트 증강  
- voxel 기반 모델(pointpillars, voxelrcnn)에서 성능 향상 확인  
- 그러나 네 모델 모두 자전거 클래스 성능 향상은 미미  
  → 객체 수가 적어 학습이 충분히 되지 않은 것으로 추측  

- 학습 및 추론 시간 감소  

- 포인트 수 기준 분석  
  - 100개 이상 포인트를 가진 객체는 증강 효과가 있음  
  - 100개 미만은 특징을 잘 잡지 못해 증강 효과 미미  

- 현재까지 실험 기반 요약본 작성 완료  

---

### 방법 정리

**Pointcloud 기반 증강**

- 입력: 눈이 내린 pointcloud  
- GT: 원본 pointcloud (raw KITTI)  

지도학습 방식으로 학습 + pointcloud에 대해서만 prediction 수행  

---

### 한계

- 눈 시뮬레이션 적용 시 포인트 유실률이 매우 큼  
- 원래 포인트 수가 적은 객체는 증강 자체가 잘 되지 않음 → 성능 한계 존재  

---

### 피드백

- GT 포인트 개수도 함께 측정  
  → GT 대비 얼마나 복원되었는지 정량 평가 필요  

- UNet 기반 포인트클라우드 복원 관련  
  → 기존 논문 참고 없이 자체적으로 진행  

- 현재 방식은 “복원”이라 보기엔 포인트 증가량이 적음  
  → 실제 성능 개선에 영향을 주는지 추가 검증 필요  
  → (시간적인 장점은 존재)  

- 추가 실험 제안  
  - GT 대비 복원된 포인트 수 정량 측정  
  - 눈 강도를 낮춘 환경에서도 실험 진행

## 3DGS

### 준비한 내용

MWFormer 날씨 이미지 복원 모델을 사용하여 날씨 아티팩트 dataset 복원 진행  

- 복원 모델의 최대 입력 해상도 제한 존재 → 처음부터 downsampling 후 실험 진행  
- 눈(snow)은 효과적으로 제거됨  
- 비(rain)는 거의 제거되지 않음  
- 눈이 길쭉한 형태일수록 제거가 잘 되지 않는 경향 존재  

---

### 해결 방향

- 다른 모델을 추가로 사용하는 방법  
- 눈(snow)에 대해서만 집중해서 진행 ✔  

---

### 참고할 점

- colmap 카메라 포즈 추정 과정에서 왜곡 보정 수행  
  → 이미지 해상도 및 비율이 미세하게 변경됨  
- 차이는 약 1~5픽셀 수준 → crop하고 평가 진행  

---

### 정량 평가 결과

- 대부분의 씬에서 전처리 모델 적용 시 성능 개선 확인  
- unposed보다 posed의 결과가 더 좋음  

- 특이 케이스  
  - hydrant: COLMAP 단계에서 해상도가 1/5가량 감소 (4/5정도가 됨)  
    → 정량평가 불가능  

- LongSplat 관련 관찰  
  - 눈이 있는 경우 오히려 정량평가 점수가 더 좋은 경우 존재  
  → 기존 경향성과 맞지 않는 결과, 서술 방식 고민 필요  

---

### 피드백

- COLMAP으로 인해 해상도가 줄어드는 경우는 일반적인 케이스가 아니므로 제외하는 것이 적절하다고 판단됨  

- 데이터셋은 LongSplat 논문에서 사용했으며, 객체 중심이 아닌 포즈 추정이 어렵게 설계된 구조  

- 그럼에도 불구하고 LongSplat보다 COLMAP + 3DGS가 더 좋은 결과를 보임  
  → 즉, LongSplat이 원래 더 강건한 모델인데 현재는 반대 결과  

- 날씨 합성 과정에서 3DGS로 먼저 재구성 후 날씨를 추가하는 방식이나, 포즈를 새로 추정했기 때문에 포즈 자체가 문제일 가능성은 낮음  

- LongSplat 논문에서도 COLMAP + 3DGS 대비 강건성을 입증함  

- 3DGS가 LongSplat보다 좋은 원인 분석 필요 (정당성 확보)  

- 전체 프로세스를 그림으로 만들어 분석 필요  

- 초록은 현재 상태 유지 가능 (두 모델 비교 중심)  

- 요약본 제출 후, 필요 시 한 모델 제거 가능  

- clean view 기반 3DGS 렌더링 결과를 GT로 사용할 필요 없이, 결과를 온전히 제시하면서 원인을 분석하는 것이 중요
