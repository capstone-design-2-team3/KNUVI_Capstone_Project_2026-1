## 기능적 요구사항

## 공통

**기상 환경 데이터 생성 및 증강 기능**

- 눈(snow), 비(rain), 야간(night) 등 기상 환경을 반영한 데이터를 생성하거나 추출할 수 있어야 한다.
- 기상 노이즈의 강도를 조절하여 다양한 실험 환경을 구성할 수 있어야 한다.

**실험 결과 분석 기능**

- 다양한 기상 환경에서 모델 성능의 변화 원인을 분석할 수 있어야 한다.
- 실험 결과를 기반으로 모델의 취약점 및 개선 방향을 도출할 수 있어야 한다.

## 3DOD 파트 (논문1, 2)

**멀티모달 3D Object Detection 모델 입력 처리**

- Camera 데이터와 LiDAR 데이터를 단일 또는 멀티모달 형태로 입력받아 처리할 수 있어야 한다.
- 증강된 기상 데이터가 기존 모델 입력 형식과 호환되도록 전처리되어야 한다.

**기상 악조건 성능 평가 기능**

- 정상 환경 대비 snow, night, rain 환경에서의 객체 탐지 성능을 평가할 수 있어야 한다.
- Bounding Box 예측 정확도 및 confidence score 변화를 측정할 수 있어야 한다.

**객체 경계 모호성 분석 기능**

- 기상 악조건에서 발생하는 객체 경계 불확실성이 모델의 탐지 성능에 미치는 영향을 분석할 수 있어야 한다.
- 센서(LiDAR, Camera)별 성능 차이를 분석할 수 있어야 한다.

**Knowledge Distillation 기반 모델 학습 기능**

- teacher 모델을 학습하고 해당 모델의 지식을 student 모델로 전달할 수 있어야 한다.
- distillation loss를 적용하여 악천후 환경에서도 강건한 3DOD 모델을 학습할 수 있어야 한다.

## 3DGS 파트 (논문3)

**Gaussian Splatting 기반 3D 재구성 기능**

- 입력 이미지 데이터를 기반으로 3D Gaussian representation을 생성하여 장면을 재구성할 수 있어야 한다.

**기상 노이즈 환경에서의 재구성 실험 및 성능 평가 기능**

- 다양한 강도의 기상 노이즈 환경에서 3D 재구성 실험을 수행할 수 있어야 한다.
- 실험 결과를 비교하여 환경 변화에 따른 성능 변화를 정량적으로 분석할 수 있어야 한다.

**비 또는 눈 환경을 고려한 하이브리드 모델 결합 기능**

- 특정 환경으로 인한 시각적 노이즈가 3D 재구성 과정에 미치는 영향을 분석할 수 있어야 한다.
- 성능 저하를 개선할 수 있도록 LongSplat과 VGGT 모델을 결합하여 기하학적 특징을 보완하는 재구성 구조를 구성할 수 있어야 한다.

## 비기능적 요구사항

**성능 및 강건성**

- 기상 악조건(눈, 비, 야간 등) 환경에서도 모델의 성능 저하를 최소화할 수 있어야 한다.
- 환경 변화로 인한 센서 노이즈나 시각적 왜곡이 존재하는 상황에서도 안정적인 객체 탐지 및 3D 재구성이 가능해야 한다.

**재현성**

- 동일한 데이터셋과 실험 환경에서 동일한 실험 결과를 재현할 수 있어야 한다.
- 실험에 사용된 모델 구조, 파라미터, 학습 설정 등을 기록하고 관리할 수 있어야 한다.

**계산 효율성**

- 모델 학습 및 추론 과정이 GPU 환경에서 효율적으로 수행될 수 있어야 한다.
- 대규모 데이터셋을 처리할 때에도 학습 및 평가 시간이 과도하게 증가하지 않도록 설계되어야 한다.

**확장성**

- 눈, 비, 야간 외에도 안개(fog) 등 다른 기상 환경에도 적용 가능하도록 구조적으로 확장 가능해야 한다.
- 다양한 3D Object Detection 및 Gaussian Splatting 모델에 적용할 수 있도록 설계되어야 한다.

**호환성**

- nuScenes 등 공개 데이터셋 구조와 호환되어야 한다.
- PyTorch 등 주요 딥러닝 프레임워크와 연동 가능해야 한다.

**분석 가능성**

- 실험 결과를 정량적 지표(예: PSNR, SSIM, 탐지 정확도 등)를 통해 분석할 수 있어야 한다.
- 다양한 기상 환경에 따른 모델 성능 변화를 비교·분석할 수 있어야 한다.
